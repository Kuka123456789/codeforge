import { Request, Response, NextFunction } from "express";
import jwt from "jsonwebtoken";

interface AuthPayload {
  userId: string;
  email: string;
  role: "admin" | "user";
}

export const authenticate = (
  req: Request,
  res: Response,
  next: NextFunction
) => {
  const token = req.headers.authorization
    ?.replace("Bearer ", "");

  if (!token) return res
    .status(401).json({ error: "Missing token" });

  try {
    const decoded = jwt.verify(
      token, process.env.JWT_SECRET!
    ) as AuthPayload;
    req.user = decoded;
    next();
  } catch {
    res.status(403).json({ error: "Invalid token" });
  }
};

Features

Everything you need to run coding agents your way

No cloud dependency. No per-seat pricing. A clean interface between you and the models you choose.

Native Slash Commands

A slash command system that works across every AI provider. Build custom workflows once, use them everywhere — no provider lock-in.

Skills Management

Organize reusable AI skills into a library your agents can invoke on demand. Extend what your coding agents can do without switching tools.

Conversation Management

Search across all chats, archive old ones, rename inline, and browse prompt history. Your conversations stay organized as they grow.

Provider Agnostic

Claude, Codex, or any supported model. Switch providers freely, bring your own API keys. Each provider remembers your preferences.

Better Git Workflow

Selective file staging, streaming git progress, and a polished commit UI. See exactly what goes into each commit before you push.

Private by Default

No telemetry, no analytics, no phone-home. Runs entirely on your machine. Your data flows directly to the AI provider you choose.

How it works

Up and running in three steps

CodeForge sits between your team and the AI provider APIs. You control the infrastructure.

01

Install CodeForge

Download the desktop app for macOS, Linux, or Windows. Or clone the repo and run from source. No containers, no complex setup.

git clone && bun install && bun dev
02

Connect your providers

Log in to your existing OpenAI or Anthropic account through the CLI. CodeForge uses your local credentials — keys never leave your machine.

03

Open your browser and code

Navigate to your server's URL, start a session, pick a model, and begin pair-programming with an AI agent. Share the URL with your team.

FAQ

Common questions

What AI providers does CodeForge support?

CodeForge supports both OpenAI Codex and Claude today, with more providers on the roadmap. You use your own API credentials and can switch between providers freely.

How do I deploy it?

Download the desktop app, or clone the repo and run bun dev. It starts the server and opens the UI in your browser. Available for macOS, Linux, and Windows.

Does my code leave my server?

CodeForge runs entirely on your infrastructure. Code context is sent to the AI provider you choose for processing, but never passes through CodeForge servers — because there are none. You host everything.

Is it free?

CodeForge is open source and free to use. The source code is fully available. A managed hosted version with team features is on the roadmap.

What's the difference vs. using the API directly?

CodeForge gives you session management, conversation history, file context, and a polished UI on top of raw APIs. It's the interface layer your team needs to use coding agents productively.
