How it works

Up and running in three steps

CodeForge sits between your team and the AI provider APIs. You control the infrastructure.

01

Deploy the server

Run CodeForge on any machine — a cloud VM, your homelab, or behind your corporate VPN. One container, no external dependencies.

docker run -p 3000:3000 codeforge/codeforge
02

Add your API keys

Plug in your existing OpenAI, Anthropic, or other provider credentials. CodeForge proxies requests through your server — keys never leave your infrastructure.

03

Open your browser and code

Navigate to your server's URL. Start a session, pick a model, and begin pair-programming with an AI agent. Share the URL with your team to give them access too.

FAQ

Common questions

What AI providers does CodeForge support?

CodeForge currently supports OpenAI Codex and is adding support for Claude and other providers. You bring your own API keys and switch between providers freely.

How do I deploy it?

Docker is the easiest way — a single container that runs the Node.js server and serves the web UI. You can also run it directly with Node.js on bare metal or any VM.

Does my code leave my server?

CodeForge runs entirely on your infrastructure. Code context is sent to the AI provider API you choose (e.g., OpenAI) for processing, but never passes through CodeForge's servers — because there are none. You host everything.

Is it free?

CodeForge is open source and MIT licensed. Free to use, modify, and self-host. A managed hosted version with team features is on the roadmap.

What's the difference between this and just using the API directly?

CodeForge gives you session management, conversation history, file context handling, and a polished UI on top of the raw APIs. Think of it as the interface layer your team needs to actually use coding agents productively.
