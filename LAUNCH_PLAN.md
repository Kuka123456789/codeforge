# CodeForge Launch Plan

## Phase 1: Validate Demand (Week 1)

- [ ] **Step 1: Landing page** — Build a simple, sharp landing page at the repo root (`/landing` or separate). Hero section, value props, demo video placeholder, email signup form.
- [ ] **Step 2: Record a demo** — 60-second screen recording showing CodeForge in action: open browser, start session, code with an agent, show the result.
- [ ] **Step 3: Deploy the landing page** — Put it live on a custom domain or Vercel/Netlify.

## Phase 2: Get Eyes On It (Week 2-3)

- [ ] **Step 4: Post on r/selfhosted** — This audience loves self-hosted dev tools. Lead with the self-hosted angle.
- [ ] **Step 5: Show HN post** — Short, honest post. "I built a self-hosted web UI for coding agents."
- [ ] **Step 6: Twitter/X thread** — Demo video + why you built it. Tag relevant people in the AI/dev tools space.
- [ ] **Step 7: Post in Discord communities** — AI engineering, self-hosted, and indie hacker servers.

## Phase 3: Talk to Users (Week 2-4)

- [ ] **Step 8: Reach out to 5-10 potential users** — Engineering managers, DevOps leads, security-conscious teams. Ask: "Would you pay for a self-hosted, provider-agnostic coding agent UI?"
- [ ] **Step 9: Collect feedback** — What features matter? What's missing? What would make them pay?

## Phase 4: Decide (Week 4)

- [ ] **Step 10: Evaluate signal** — How many signups? What did conversations reveal? Is there a paying audience?
- [ ] **Step 11: If yes** — Build team features (auth, usage tracking, shared sessions). Set up a hosted tier.
- [ ] **Step 12: If no** — Pivot the angle based on feedback, or keep it as a strong open-source project.
